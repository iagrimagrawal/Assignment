Here's a table comparing some of the key differences between HTML and HTML5:

Features HTML HTML5
DOCTYPE Strict, Transitional, or Frameset doctype declaration Single doctype declaration:DOCTYPE html
Semantic Tags Limited semantic tags (e.g., div, span) Extensive set of semantic tags (e.g.,header, nav, article)
Audio/Video No built-in support for audio/video playback Native support for audio and video elements
Form Elements Basic form elements (e.g., input, textarea, select) Enhanced form elements (e.g., input type="date", input type="email")
Accessibility Limited accessibility features Improved accessibility support (e.g., ARIA attributes)
Canvas Not available Introduction of canvas element for drawing graphics
Geolocation Not available Geolocation API for obtaining user's location
Offline Support Not available Offline Web Application API for creating offline-capable web apps
Web Workers Not available Web Workers API for running scripts in the background
Local Storage Not available Local Storage API for client-side storage

It's important to note that HTML5 is an evolution of HTML and includes many new features and improvements over its predecessor. The introduction of semantic tags, enhanced form elements, audio/video support, and various APIs make HTML5 more robust and suitable for modern web development.
